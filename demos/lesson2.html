In WebGL, pictures are created by drawing points, lines, triangles, etc. on the screen. With Facet, these are packaged in models, which are created with Facet.model. Each model is created by specifying the type of primitive being drawn (type), a list of vertex coordinates and their dimension (vertex), and the sequence in which these vertices are drawn (elements):

var square = Facet.model({
    type: "triangles",
    elements: [0, 1, 2, 0, 2, 3],
    vertex: [[-1,-1, 1,-1, 1,1, -1,1], 2]
}), triangle = Facet.model({
    type: "triangles",
    elements: 3, // this is equivalent to [0, 1, 2]
    vertex: [[0,1, -1,-1, 1,-1], 2]
});

To draw each model, we need to tell WebGL how to give a position to each vertex, and a color to each pixel that gets drawn on the screen. These are created in Facet with Shade, and the expressions are baked into drawables.

In WebGL and OpenGL, vertices in three-dimensional space are given in 4 dimensions, in homogeneous coordinates. We put the square and triangle in 3D homogeneous space by picking x and y from their vertex attribute, setting z to be zero and the fourth coordinate to be 1. Variables with values that do not change as the model is being drawn are called uniforms, and here we specify a uniform mvp which will store a 4x4 matrix. This variable stores the model-view-projection matrix, the transformation of a vertex from its position "in the world" to the position "inside the camera". The final vertex position is given by multiplying the vertex coordinates by that matrix:

mvp = Shade.uniform("mat4");

square_drawable = Facet.bake(square, {
    position: mvp.mul(Shade.vec(square.vertex, 0, 1)),
    color: Shade.color('white')
});

triangle_drawable = Facet.bake(triangle, {
    position: mvp.mul(Shade.vec(triangle.vertex, 0, 1)),
    color: Shade.color('white')
});

We are almost done. We now create a display function. Drawables have a method predictably named draw, which draws the models using the given positions and colors. The camera transformation is computed by Facet.perspective, and the models are positioned using Facet.translation: 

function draw_it()
{
    var proj = Facet.perspective(45, 720/480, 0.1, 100.0);

    mvp.set(mat4.product(proj, Facet.translation( 1.5, 0.0, -6.0)));
    square_drawable.draw();

    mvp.set(mat4.product(proj, Facet.translation(-1.5, 0.0, -6.0)));
    triangle_drawable.draw();
}

In this last snippet of code we create the required WebGL scaffolding, using Facet.initGL. Notice that Facet populates the WebGL context with an extra function, display. This function does a few convenient things such as clearing the color and depth buffers, and then calls the display function you give it, in this case draw_it. 

var canvas = document.getElementById("webgl");
gl = Facet.initGL(canvas, {
                      clearDepth: 1.0,
                      clearColor: [0,0,0,1],
                      display: draw_it,
                      attributes: {
                          alpha: true,
                          depth: true
                      },
                      debugging: true
                  });
gl.display();

That's it! Your full-fledged WebGL app is up and running.
