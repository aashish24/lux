WebGL objects cannot be shared across contexts, which mean that when two different WebGL contexts are active at the same time, one must usually be careful to call the appropriate WebGL functions with the appropriate context object. Facet does the multiple-context book-keeping mostly transparently for you.

Note, however, that some Shade objects have context-sensitive WebGL objects associated with it. Those Shade expressions must only be used with other objects from the same context.

In addition, if you issue an asynchronous call (typically an AJAX request), there is generally no guarantee that the invoked callback will have the same current active context. Facet.Net.* calls set the right context on callback, but if you use asynchronous routines (requestAnimFrame, for example), you'll have to be careful about it.
